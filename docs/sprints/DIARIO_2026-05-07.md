## Causa raíz del incidente del día

El bug de "botones de inicio de chequeo no funcionan" reportado por Jorge
**no era un bug de código** — el código del repo estaba correcto desde
SPRINT-103 (`1568a63`, 2026-05-06). La causa fue **operativa**: las rules
con `.get(field, null)` se commitearon y pushearon a main, pero nadie
ejecutó `npm run deploy:rules`. Vercel deploya el frontend automático con
cada push, pero las rules de Firestore requieren un comando manual
separado. Resultado: el código nuevo (que asume rules nuevas) chocó con
rules viejas en producción durante ~24h.

Cualquier técnico que intentaba "Iniciar chequeo" en una orden regular
recibía permission-denied silencioso porque la rule vieja evaluaba
`request.resource.data.soloChequeo == resource.data.soloChequeo` con
ambos lados `missing` → la igualdad fallaba (Firestore Rules no resuelve
`null == null` con acceso directo, sólo con `.get(field, null)`).

## Capa nueva instalada hoy para que NO se repita

1. **`scripts/invariantes/check-rules-pendientes-deploy.ts`** — cazador
   P-005. Calcula SHA-256 de `firestore.rules` y lo compara contra
   `firestore.rules.deployed.lock` (un archivo nuevo que se actualiza
   automáticamente después de cada `npm run deploy:rules`). Si los hashes
   difieren, **bloquea el pre-commit hook** con un mensaje explícito que
   dice "ejecutá `npm run deploy:rules` para sincronizar".

2. **`package.json` script `deploy:rules` ahora es compuesto:**
   `firebase deploy --only firestore:rules && tsx
   scripts/invariantes/marcar-rules-deployadas.ts`. Imposible deployar
   sin que el lock se actualice. Si alguien ejecuta `firebase deploy`
   directo (fuera del npm script), el cazador grita en el siguiente
   commit hasta que se sincronice manualmente con `npx tsx
   scripts/invariantes/marcar-rules-deployadas.ts`.

3. **Sub-regla obligatoria nueva en CLAUDE.md:** "Sprints que tocan
   `firestore.rules` deben ejecutar `npm run deploy:rules` ANTES de
   cerrar COMPLETADO." Antiprecedente SPRINT-103/106 documentado.

4. **Sub-regla colateral en CLAUDE.md:** cleanup de dead code en páginas
   críticas requiere QA manual del flujo afectado antes de commit.
