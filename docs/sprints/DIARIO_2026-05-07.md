## Causa raíz del incidente del día

El bug de "botones de inicio de chequeo no funcionan" reportado por Jorge
**no era un bug de código** — el código del repo estaba correcto desde
SPRINT-103 (`1568a63`, 2026-05-06). La causa fue **operativa**: las rules
con `.get(field, null)` se commitearon y pushearon a main, pero nadie
ejecutó `npm run deploy:rules`. Vercel deploya el frontend automático con
cada push, pero las rules de Firestore requieren un comando manual
separado. Resultado: el código nuevo (que asume rules nuevas) chocó con
rules viejas en producción durante ~24h.

Cualquier técnico que intentaba "Iniciar chequeo" en una orden regular
recibía permission-denied silencioso porque la rule vieja evaluaba
`request.resource.data.soloChequeo == resource.data.soloChequeo` con
ambos lados `missing` → la igualdad fallaba (Firestore Rules no resuelve
`null == null` con acceso directo, sólo con `.get(field, null)`).

## Capa nueva instalada hoy para que NO se repita
