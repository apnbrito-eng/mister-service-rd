## Pasada 5 — SPRINT-WA-1-HOTFIX FieldValue import (commit `50931bb`)

**Bug en producción descubierto por agente devops en verificación post-deploy.**

Endpoint `/api/whatsapp/webhook` devolvía HTTP 500 FUNCTION_INVOCATION_FAILED en TODOS los métodos porque crasheaba en module-load.

**Causa raíz:** `import { FieldValue, getAdminFirestore } from '../_lib/firebaseAdmin.js'` intentaba importar `FieldValue` desde un archivo que NO la exporta. firebaseAdmin.ts usa FieldValue internamente (línea 5 + 177) pero nunca la re-exporta.

**Por qué no lo cazó el typecheck pre-commit:** `tsconfig.json` del proyecto incluye únicamente `"src"`. El directorio `api/` se compila aparte por Vercel al deploy time. Resultado: typecheck "verde" pero `api/` desnudo.

**Fix de 2 líneas:**
```diff
- import { FieldValue, getAdminFirestore } from '../_lib/firebaseAdmin.js';
+ import { FieldValue } from 'firebase-admin/firestore';
+ import { getAdminFirestore } from '../_lib/firebaseAdmin.js';
```

**Postmortem generado:** `docs/postmortems/2026-05-19-wa-1-webhook-fieldvalue-import.md`.

**Acción preventiva propuesta (no implementada en este sprint, sprint follow-up):** crear `tsconfig.api.json` con `target: ES2022`, `module: NodeNext`, `include: ["api"]` + agregar al pre-commit hook step `tsc -p tsconfig.api.json --noEmit`. Sin esto, el próximo bug de imports en `api/` se repetirá.
